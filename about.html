About ScreenGuard

ScreenGuard is a web-based screen time control application built as a college mini project. It helps users monitor app usage, set smart time limits, build healthier digital habits, and boost productivity through focused work sessions.

💡 Study & Focus Tips
Science-backed strategies to improve concentration and manage your digital habits
🍅

The Pomodoro Technique

Work in focused 25-minute bursts followed by a 5-minute break. After 4 sessions, take a 15–30 minute long break. This prevents mental fatigue and keeps your brain sharp throughout the day.

Productivity
📵

Phone-Free Study Zones

Research shows that just having your phone visible (even face-down) reduces your available cognitive capacity. Place it in another room or inside a drawer during study sessions.

Habit
🌅

No Screens for the First Hour

Avoid checking your phone for the first 60 minutes after waking up. This helps your brain enter a calm, focused state before being bombarded with notifications and social media.

Morning Routine
🌙

Screen-Free 1 Hour Before Bed

Blue light from screens suppresses melatonin production and delays sleep by up to 90 minutes. Set a screen cutoff time in ScreenGuard's Quiet Hours to build this habit automatically.

Set a Daily Screen Time Goal

Students who track their screen time and set a daily limit reduce their usage by an average of 40% within two weeks. Use ScreenGuard's daily goal setter in Profile → Usage Goals.

Use Do Not Disturb During Study

Every notification interruption takes an average of 23 minutes to fully recover focus from. Enable ScreenGuard alerts only for critical events, and silence your device during focus sessions.

The 20-20-20 Eye Rule

Every 20 minutes of screen use, look at something 20 feet away for 20 seconds. This relaxes your eye muscles, reduces digital eye strain and headaches during long study sessions.

❓ Frequently Asked Questions
Quick answers to common questions about the project
🔐 Can I use my real Google or Apple account to log in? ▼

Not in this version. The Google and Apple login buttons are UI-simulated to demonstrate the interface design. Real OAuth login requires a registered backend server and API credentials from Google Cloud Console or Apple Developer Program. For this college mini project, the login UI demonstrates knowledge of authentication flows without requiring a live server.

📱 Does ScreenGuard actually monitor my phone's apps? ▼

No — this is a front-end demonstration project. Real device-level app monitoring requires OS-level permissions (Android's UsageStatsManager or iOS Screen Time API) and a native mobile app or browser extension. The usage data shown is simulated to demonstrate the UI, monitoring logic, chart visualisations, and alert systems that a real app would use.

💾 Does the app save my data when I close the browser? ▼

Partially. Your dark/light mode theme preference is saved using localStorage and persists between sessions. However, usage data, app limits you set, and notification history are held in memory (JavaScript variables) and will reset when you close or refresh the page. A real version would use a database like Firebase or MongoDB to persist all data.

🛠️ What tools do I need to run this project? ▼

Just two things:

  • Visual Studio Code — free code editor from Microsoft (code.visualstudio.com)
  • Live Server extension — install from VS Code's Extensions tab (by Ritwick Dey)

No Node.js, npm, or any other installation is required. All libraries (Chart.js, Google Fonts) are loaded via CDN links.

🚀 Can this project be extended into a real app? ▼

Absolutely. To turn this into a full production app you would need to:

  • Add a backend (Node.js + Express or Python + Flask) to store user data
  • Connect a database (Firebase, MongoDB, or PostgreSQL)
  • Implement real OAuth via Google Cloud Console
  • Build a browser extension to actually track tab/app usage
  • Deploy to a hosting service like Vercel, Netlify, or Render
